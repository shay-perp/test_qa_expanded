| 10 | `Ammeters/base_ammeter.py` | 22 | Infinite `while True` loop with no way to stop the server thread | `while True:` | `while not self._stop_event.is_set():` | Threads could never be joined; program could not exit cleanly; zombie threads held ports open |

| 12 | `Ammeters/base_ammeter.py` | 18 | `socket` used as context manager with `with`; a timeout on `accept()` was not possible without restructuring | `with socket.socket(...) as s:` | Explicit `s = socket.socket(...)` / `try…finally s.close()` with `s.settimeout(SOCKET_TIMEOUT_SEC)` | `accept()` must time out periodically so the stop-event check in the loop can be reached |
| 13 | `Ammeters/base_ammeter.py` | 23 | No timeout set on accepted connection socket | *(absent)* | `conn.settimeout(SOCKET_TIMEOUT_SEC)` | Malformed or slow clients could block `recv()` forever, hanging the server thread |

| 17 | `Ammeters/Circutor_Ammeter.py` | 5-9 | Same as above for CIRCUTOR (plus the extra suffix bug) | `return b'MEASURE_CIRCUTOR -get_measurement -current'` | `return self._command.encode('utf-8')` | Removes both the hardcoding and the wrong `-current` suffix in one fix |
| 18 | `main.py` | 11-20 | Ports and commands scattered as hardcoded literals | `GreenleeAmmeter(5001)`, `time.sleep(5)` | All values sourced from `config_loader` + `constants.py`; sleep uses `SERVER_STARTUP_SEC` | Hardcoded magic numbers are fragile; a single config change now propagates everywhere automatically |
